## 修复群聊回复 bug
- 问题: 网站群聊@bot后，WorkerClaw能收到并回复，但网站群聊收不到回复
- 根因: 服务端有两个 WebSocket 端点——Chat WebSocket（聊天室专用）和 OpenClaw WebSocket（`/ws/openclaw`）

- 修复: 在 OpenClaw WebSocket handler 中增加 `case 'chat'` 处理，复用聊天室逻辑（频率限制→保存DB→广播给chatClients和openclawClients）

## 修复 WorkerClaw 私信回复 401 "请先登录"

### 问题
- WorkerClaw 收到私信后，LLM 生成回复，但 `PlatformAPI.sendPrivateMessage()` 返回 `请先登录` (401 AUTH_REQUIRED)

### 根因
- `sendPrivateMessage()` 等方法只发了 `Authorization: Bearer ${token}`，但 agent 的 token 是 raw token 不是 JWT
- 服务端 `jwtAuthMiddleware` JWT 验证失败后，走 agent fallback 需要 `X-Bot-Id` + `X-Bot-Token` header
- WorkerClaw 没有发送这两个 header → 401

### 修复
1. **`request()` 统一方法** (platform-api.ts): 增加 `X-Bot-Id` 和 `X-Bot-Token` header（与 Bearer 并行发送）
2. **直接用 fetch() 的方法** (sendPrivateMessage/postComment/cancelTake/applyArbitration/sendChatMessage): 全部改为使用 `this.request()` 统一方法
3. **服务端双保险** (server.js jwtAuthMiddleware): agent fallback 的 token 获取从 `req.headers['x-bot-token']` 扩展为也支持 `req.body.token`
4. 版本升级到 v0.10.3，已发布 npm 并部署到 Docker

## 修复 profile.html 发私信被踢出登录

### 问题
- 塘主用户在 profile.html 点击发私信，发一条就被踢回登录页
- 所有 POST 操作（私信、关注等）都会触发 401 → 全局 fetch 拦截器自动跳转 login.html

### 根因
- `isLoggedIn()` 只检查 `SESSION_KEY`/`BOT_ID_KEY`/`PRIVATE_KEY_KEY`，不检查 JWT
- 塘主用户的 JWT 可能过期（7天有效）或从未被存储（localStorage 被清理等）
- POST 请求全局拦截器自动注入 `Authorization: Bearer ${jwt}`，但 JWT 为 null 时裸 POST → 服务端 401
- 401 响应 `code: 'AUTH_REQUIRED'` → 全局拦截器清除 session 并跳转登录

### 修复 (session.js)
- 新增 `refreshJWTWithPrivateKey()` 函数：用 localStorage 中的私钥重新 RSA 签名，调用 `/api/login` 获取新 JWT
- 修改全局 fetch 拦截器：遇到 401 + AUTH_REQUIRED 时，先尝试自动刷新 JWT，成功后自动重试原请求
- 刷新失败才跳转登录页
- 加了并发锁防止多个 401 同时触发多次刷新
