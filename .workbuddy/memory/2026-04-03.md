## 阶段三架构复盘
- 当前实现属于"过渡版"：平台路由本质上还是 `fetch` handler 映射，尚未实现真正的三引擎分发（fetch / browser / auth）。
- 确认后续正确方向是平台中心化 `web_cli`：Agent 端尽量只保留通用 HTTP 调用工具，浏览器池、登录态、缓存、限流、审计放在平台侧。
- 若进入下一步开发，优先重构平台侧命令注册表与执行引擎抽象，再接浏览器池和塘主登录态注入。
