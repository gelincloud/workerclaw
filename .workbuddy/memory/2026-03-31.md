# 2026-03-31 工作记录

## WorkerClaw v0.13.5 修复

### 问题分析
用户报告任务"访问baidu.com并截图"失败，日志显示两个问题：

1. **浏览器导航失败**：`Cannot read properties of undefined (reading 'length')`
   - 原因：`page.evaluate(extractFn)` 返回 `undefined`，代码直接访问 `.length`
   - 位置：`browser-sandbox.ts` 第 250-254 行

2. **LLM 请求超时**：GLM 模型响应慢，两次重试都超时（默认 120s）

### 修复内容
- `src/sandbox/browser-sandbox.ts`：添加 `|| ''` 默认值处理 `content` 为 `undefined` 的情况

### 发布
- `workerclaw@0.13.5` 已发布到 npm

## WorkerClaw v0.13.6 修复

### 问题分析
用户反馈 LLM 超时频繁，发现根本原因：
- `llm-provider.ts` 中三个适配器（OpenAI/Claude/Gemini）的超时都是**硬编码** `120_000`
- 配置中的 `llmTimeoutMs` 完全被忽略，从未生效

### 修复内容
1. 三个适配器都改为读取 `config.timeout.llmTimeoutMs`
2. 默认超时：60s → **180s (3分钟)**
3. 超时重试：`retryOnTimeout: false` → `true`
4. 重试次数：1 → **2**
5. 重试间隔：5s → **3s**
6. 配置传递：`task.timeout.llmTimeoutMs` 自动传递到 `llm.timeout`

### 发布
- `workerclaw@0.13.6` 已发布到 npm
